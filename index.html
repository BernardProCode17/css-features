New CSS Technologies

We'll be covering three CSS properties that are now widely supported by modern browsers. These properties give you greater control over your website elements and help you create visually appealing and responsive content.

These CSS properties are:

  • <accent-color>
  • <dynamic viewport>
  • <container query>

1. Accent color

Supported Browsers

What is accent color?

The accent color property allows you to add color to HTML form inputs such as checkboxes, radio buttons, and progress bars.

  • <input type="checkbox">
  • <input type="radio">
  • <input type="range">
  • <progress>

Why use accent color?

Color plays a key role in defining your brand; therefore, tools that enable you to apply color styling to all elements of your website are of great value.

Accent color makes it easy to manage the look of forms on your website so that they match your brand’s color theme.

  • Saves you time

    • Before accent color, if you didn’t like the browser’s default style, your other option was to recreate the form inputs from scratch, which can be time-consuming and complex.

  • Built-in accessibility

    • Depending on what color you pick to style the inputs, the browser will automatically set the input background to either white or black to provide optimal color contrast.

2. Dynamic Viewport

Supported Browsers

What is dynamic Viewport?

CSS viewport units - ‘vw’ for width and ‘vh’ for height - work well on desktop, but on mobile devices. They do not take into consideration the dynamic UI elements such as the address or the navigation bar.

That means when these UI elements expand or retract, your content may not display as intended. Dynamic viewport properties provide a way for you to get the viewport dimensions as specified.

Why use Dynamiv ViewPort?

On mobile, the address bar at the top and the navigation bar at the bottom, typically changes in size as the user scrolls up or down. But it does not changes the viewport to respond to the newly available space on the device.

That why we have the new viewport units to make the viewport respond to the full specified dimensionso f the viewport.

Dynamic Viewport properties

  • svw
  • svh
  • lvw
  • lvh
  • dvw
  • dvh

There are 3 new units for the dynamic viewport properties. SV, LV DV, with their respective width and height. In most cases you would use the DV as it allows for responsiveness and is between the SV LV.

SV sets the size of the viewport with the address and navigation bar in consideration, while LV does not consider the address and navigation bar. With DV it acts as both the SV and the LV, making the viewport update responsively.

3. Container Queries

Supported Browsers

What is Container Query

Container query is a CSS propery, that allows you to adjust the style of an element based on its container dimensions and not the viewport. (Mozilla Developer Network, n.d.).

With a container query, you would use the following properties and units:

  1. Container-type:

    This is a property that helps the browser know which containers you might be interested in checking later for their size. The values that can be used for this property are size, inline-size, or normal (Mozilla Developer Network, n.d.).

  2. Container-name:

    This is another way to give a containment context. Once the container is named, the name can be used in a container query to target a specific container (Mozilla Developer Network, n.d.).

  3. Inline-size:

    This refers to the width of the container. It considers how wide something is and affects how elements are laid out horizontally. It controls how the content fits and looks in that width (Mozilla Developer Network, n.d.).

  4. Size:

    This considers both the width and the height of a container where content is placed. It deals with how elements are arranged and how large the container holding the content is. It affects how the layout, appearance, and overall size of the container are controlled (Mozilla Developer Network, n.d.).

  5. Normal:

    This means that the element does not dictate or set any rules specifically for the size of its container. It does not play a role in queries about container size, but it affects the style and appearance of the container itself (Mozilla Developer Network, n.d.).
  • cqw: 1% of the query container width
  • cqh: 1% of the query container height
  • cqi: 1% of a query conntainer inline size
  • cqb: 1% of a query container block size
  • cqmin: The smaller value of either cqi or cqb
  • cqmax: The lager value of either cqi or cqb

Why use container Query

Specificity

container query can help you target elements with specific styles at a set dimension without media query. Previously, if you wanted to change an element based on the device dimensions you would use a media query, which is based on the DOM viewport dimensions. That would mean you need to make a new media query to change that element styling when the viewport changes.

Responsiveness

With container query you can write a query for that "Container" and when the viewport changes the styles will apply to the element as specified, without having to change other elements or writing a new media for that container or other elements/containers.

customization

It allows for more customization options. For example, you may want to modify a sidebar, at a certain width or height you can set a query to sticky the sidebar at set dimenions, stack in a column below set width, or removed at set height, without having to styles other elements that is not affected by the viewport size changes.

Where would you use a Container Query

Container query can be used wherever you need to adjust an element’s style or lay based on its container. It is Kind of an alternative to a media query, but best used with media queries as they work in relation to each other.

Container queries works like flex and grid, where the styles is applied to the descendant element of a container. Such as blocks level elements like sections, articles, cards, forms, and sidebars and more. Additionally, it could be used for global blocks such as header, footer, and main.
